Core Concepts

8 topics

Repository

Repo
git init

The project folder Git tracks. Stores the complete version history, all branches, commits, and metadata in a hidden .git directory.

Best Practices

  • Use .gitignore to exclude secrets and build artifacts
  • One focused project per repository
  • Set a clear branch strategy from the start

Commit

Snapshot
git commit -m "feat: add login"

A permanent snapshot of staged changes saved with a message. Each commit links to its parent, forming an unbreakable chain of project history.

Best Practices

  • Write messages in imperative mood: Add feature
  • Commit small, logical units of work
  • Stage only related files per commit

Branch

Parallel
git checkout -b feature/auth

A lightweight pointer to a commit. Branches isolate your work so features, bug fixes, and experiments never interfere with each other.

Best Practices

  • Name branches clearly: feature/, fix/, release/
  • Keep branch lifetimes short to minimize conflicts
  • Sync with main regularly

Merge

Combine
git merge feature/x

Integrates changes from one branch into another, creating a merge commit that preserves the full history of both lines of development.

Best Practices

  • Merge frequently to keep diff sizes small
  • Use --no-ff to keep branch history visible
  • Always review changes before merging

Rebase

Rewrite
git rebase main

Replays your branch's commits onto a new base, rewriting history so it looks like your work always started from the latest main.

Best Practices

  • Never rebase commits already pushed to a shared branch
  • Use interactive rebase to clean up local history
  • Resolve conflicts carefully at each replayed commit

Push / Pull

Sync
git push origin main

Push uploads local commits to the remote repository. Pull downloads remote changes and merges them into your current branch.

Best Practices

  • Pull before pushing to reduce merge conflicts
  • Use git fetch to preview remote changes first
  • Never force-push to shared branches

Pull Request

PR
gh pr create --title "Add login"

A request to review and merge your branch into another. The hub of code collaboration — discussions, approvals, and CI checks happen here.

Best Practices

  • Write a clear description and link related issues
  • Use draft PRs for work in progress
  • Keep PRs small for faster, better reviews

Merge Conflict

Conflict
git mergetool

Occurs when two branches edit the same lines differently. Git pauses the merge and marks both versions — you decide what the final code looks like.

Best Practices

  • Understand both sides before resolving
  • Use a visual merge tool to compare side-by-side
  • Test thoroughly after resolving a conflict
